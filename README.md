# KiloCode Custom Modes

Данный проект содержит различные пользовательские режимы для расширения функциональности KiloCode. Каждый режим представляет собой отдельный модуль, который может быть загружен для предоставления специфических возможностей или языковых компонентов. Проект включает в себя восемь основных режимов: System Analyst Pro, Orchestrator, Project Manager, TG Bot Expert, Code, Universal Coding Agent, Excel VBA Coder, Keenetic & Synology Admin и Moscow Estate Expert, каждый из которых предоставляет уникальные возможности для различных задач разработки, администрирования, юриспруденции и управления.

## Назначение проекта

Этот проект служит репозиторием для пользовательских режимов в KiloCode, расширяя его функциональность за счет специализированных модулей. Каждый пользовательский режим предоставляет определенные возможности или языковые компоненты, которые улучшают основной опыт работы с KiloCode.

## Постановка проблемы

Пользователям KiloCode требуются специализированные режимы для различных типов работы, выходящие за рамки основной функциональности. Этот проект решает потребность в модульных, многократно используемых расширениях, которые можно загружать по мере необходимости для предоставления специфических для домена возможностей.

## Решение

Проект предоставляет структурированный подход к разработке и организации пользовательских режимов в виде отдельных модулей, которые можно легко интегрировать в KiloCode. Каждый режим автономен и имеет собственную конфигурацию и возможности.

## Целевая аудитория

- Пользователи KiloCode, которым требуется специализированная функциональность
- Разработчики, создающие пользовательские режимы
- Организации, нуждающиеся в специфических инструментах для работы с KiloCode

## Ключевые преимущества

- Расширяемость: позволяет расширять KiloCode дополнительной функциональностью
- Модульность: каждый режим независим и может разрабатываться отдельно
- Повторное использование: режимы могут использоваться и переиспользоваться в разных контекстах
- Организация: централизованный репозиторий для пользовательских режимов
- Координация: режим Orchestrator обеспечивает управление сложными рабочими процессами
- Стратегическое планирование: режим Project Manager поддерживает планирование бюджета и сроков
- Разработка Telegram-ботов: режим TG Bot Expert предоставляет специализированные инструменты для разработки ботов с интеграцией GitLab CI/CD
- Универсальная разработка: режимы Code и Universal Coding Agent обеспечивают поддержку всех языков программирования и фреймворков
- Специализированная разработка Excel VBA решений: режим Excel VBA Coder предоставляет инструменты для создания надежных Excel-приложений корпоративного уровня
- Администрирование сетевого оборудования: режим Keenetic & Synology Admin обеспечивает экспертизу в области кибербезопасности и администрирования сетей на базе роутеров Keenetic и NAS-устройств Synology
- Юридическая экспертиза недвижимости: режим Moscow Estate Expert предоставляет юридическую поддержку по вопросам московской недвижимости, переоборудования и земельного права

## Технологии и интеграции

### Основные технологии
- YAML: Используется для файлов конфигурации режимов (например, system_analyst_mode-v1.0.yaml, system_analyst_mode-v1.1.yaml, orchestrator-export.yaml, Project_manager.yaml, TG-Bot-Expert.yaml, code-original.yaml, universal-coding-agent.yaml, keenetic-synology-admin.yaml, MoscowEstateExpert.yaml, universal-coding-agent_v1-1.yaml, universal-coding-agent_v1-2.yaml, universal-coding-agent_v1-3.yaml)
- Markdown: Основной формат документации во всем проекте
- Git: Система контроля версий с комплексной конфигурацией .gitignore

### Структура проекта и конфигурация
- **Определение режимов**: Пользовательские режимы определяются с использованием YAML-файлов конфигурации
- **Контроль доступа к файлам**: Система детализированных разрешений, ограничивающая доступ к редактированию определенных типов файлов для каждого режима
- **Стандарты документации**: Структурированный подход к технической документации
- **Поддержка нескольких языков**: Режимы могут быть реализованы с языковыми вариантами (ENG/RUS)
- **Языковые правила**: Поддержка специфических правил для разных языков (например, русский язык в `language-rules/RU/language_ru.md`)

## Установка и использование

### Установка

1. Клонируйте репозиторий в локальную папку:
   ```bash
   git clone https://github.com/ваш-репозиторий/kilocode-custom-modes.git
   ```

2. Скопируйте нужные файлы режимов в ваш проект или глобальную директорию KiloCode.

3. Для каждого режима, который вы хотите использовать, убедитесь, что соответствующий YAML-файл конфигурации добавлен в настройки KiloCode.

### Использование

1. Откройте проект в VSCode с установленным расширением KiloCode.

2. Перейдите в настройки KiloCode и добавьте пути к YAML-файлам конфигурации нужных режимов.

3. Выберите нужный режим из выпадающего списка в интерфейсе KiloCode.

4. Каждый режим будет работать в соответствии с его спецификацией:
