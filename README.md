# KiloCode Custom Modes

Данный репозиторий представляет собой коллекцию независимых проектов, каждый из которых реализует пользовательский режим для расширения функциональности KiloCode. Каждый каталог в этом репозитории - это отдельный автономный проект, предоставляющий специфические возможности или языковые компоненты для работы с KiloCode.

## Назначение проекта

Каждый каталог в этом репозитории содержит отдельный проект, представляющий собой пользовательский режим для KiloCode. Эти режимы позволяют расширить функциональность KiloCode за счет специализированных модулей, каждый из которых решает конкретные задачи в своей области.

## Постановка проблемы

Пользователям KiloCode требуются специализированные режимы для различных типов работы, выходящие за рамки основной функциональности. Этот репозиторий решает потребность в модульных, многократно используемых расширениях, которые можно загружать по мере необходимости для предоставления специфических для домена возможностей.

## Решение

Репозиторий предоставляет структурированный подход к разработке и организации пользовательских режимов в виде отдельных автономных проектов, которые можно легко интегрировать в KiloCode. Каждый проект автономен и имеет собственную конфигурацию и возможности.

## Описание проектов

### System Analyst Pro

Расположен в директории `system_analyst/`, основная конфигурация в файлах `system_analyst_mode-v1.0.yaml` и `system_analyst_mode-v1.1.yaml`.

**Ключевые особенности:**
- Полнофункциональные возможности IT-анализа
- Инженерия требований (SRS/Пользовательские истории)
- Моделирование бизнес-процессов (BPMN)
- Проектирование API (REST/SOAP)
- Архитектура данных (SQL/NoSQL)
- Системная интеграция

**Технические возможности:**
- Сбор и документирование требований
- Моделирование бизнес-процессов с BPMN 2.0
- Создание UML-диаграмм (Классов, Последовательности, Деятельности, Машины состояний)
- Визуализация архитектуры по модели C4
- Проектирование схем баз данных и создание ERD
- Проектирование спецификаций API (OpenAPI/Swagger)
- Генерация SQL-запросов с расширенными возможностями (CTE, Оконные функции)

**Технологии:**
- YAML: используется для файлов конфигурации режимов
- Markdown: формат документации
- PlantUML и Mermaid.js: для визуального моделирования
- SQL: для проектирования баз данных
- OpenAPI/Swagger: для проектирования API

## Установка и использование

### Установка

1. Клонируйте репозиторий в локальную папку:
   ```bash
   git clone https://github.com/ваш-репозиторий/kilocode-custom-modes.git
   ```

2. Скопируйте нужные файлы режимов в ваш проект или глобальную директорию KiloCode.

3. Для каждого режима, который вы хотите использовать, убедитесь, что соответствующий YAML-файл конфигурации добавлен в настройки KiloCode.

### Использование

1. Откройте проект в VSCode с установленным расширением KiloCode.

2. Перейдите в настройки KiloCode и добавьте пути к YAML-файлам конфигурации нужных режимов.

3. Выберите нужный режим из выпадающего списка в интерфейсе KiloCode.

4. Каждый режим будет работать в соответствии с его спецификацией:
   - **System Analyst Pro**: Для анализа требований, проектирования API, создания диаграмм
   - **Orchestrator**: Для координации сложных рабочих процессов
   - **Project Manager**: Для управления проектами и расчета бюджетов
   - **TG Bot Expert**: Для разработки Telegram-ботов с интеграцией GitLab CI/CD
   - **Code/Universal Coding Agent**: Для универсальной разработки на различных языках
   - **Excel VBA Coder**: Для разработки Excel-приложений с VBA
   - **Keenetic & Synology Admin**: Для администрирования сетевого оборудования
   - **Moscow Estate Expert**: Для юридических консультаций по недвижимости в Москве

### Система контроля доступа

Каждый режим реализует детализированный доступ:
- Доступ на чтение ко всем файлам
- Доступ к браузеру для веб-операций
- Доступ на редактирование ограничен определенными типами файлов в зависимости от режима:
  - **System Analyst Pro**: Документация (`.md`, `.mdx`, `.txt`), Конфигурация (`.json`, `.yaml`, `.yml`, `.xml`), Форматы данных (`.sql`, `.csv`, `.tsv`), Спецификации диаграмм (`.puml`, `.mermaid`, `.graphql`), Форматы тестирования (`.feature`, `.drawio`)
  - **Project Manager**: Документы управления проектами (`.md`, `.docx`, `.xlsx`, `.xls`, `.csv`, `.pdf`, `.json`, `.yaml`, `.yml`)
  - **Orchestrator**: Нет специальных ограничений на редактирование файлов (координирует другие режимы)
  - **TG Bot Expert**: Полный доступ к редактированию всех типов файлов для комплексной разработки ботов
  - **Keenetic & Synology Admin**: Полный доступ к редактированию конфигурационных файлов и документации для администрирования сетевого оборудования
  - **Moscow Estate Expert**: Полный доступ к редактированию юридической документации и справочных материалов
  - **Code**: Полный доступ к редактированию всех типов файлов для универсальной разработки
  - **Universal Coding Agent**: Полный доступ к редактированию всех типов файлов для универсальной разработки
