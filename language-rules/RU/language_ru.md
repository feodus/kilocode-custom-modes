# Language Rule: Russian Language

## Primary Requirement
All AI agents MUST use exclusively Russian language for communicating with the user.

## Exceptions
The following elements MAY use Latin characters or English language by their nature:
- Technical terminology (e.g., API, JSON, HTTP, SQL)
- Code and programming syntax constructs
- Variable, function, class names and other software entities
- Code formatting and data structures
- Names of libraries, frameworks and tools
- Technical designations and specifications
- Standards and protocols (e.g., REST, SOAP, OAuth)
- CLI commands and parameters
- Identifiers, tags and labels (e.g., HTML tags, CSS selectors)
- Logs and debugging information
- Formulas and mathematical expressions
- File names and extensions
- Paths and addresses (URL, file paths)
- Specifications and data schemas

## Examples of Correct Usage
### Correct:
- "Для работы с API используйте метод GET"
- "Создайте переменную `userCount` типа integer"
- "Формат JSON должен быть валидным: {\"name\": \"value\"}"

### Incorrect:
- "Я могу помочь вам с this задачей" (mixed language)
- "Давайте сделаем setup проекта" (unjustified use of English)

## Purpose
To ensure consistent and clear communication in Russian language while maintaining precision in technical discussions.
