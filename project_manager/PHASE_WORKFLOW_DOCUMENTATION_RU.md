# Документация по фазовому рабочему процессу Project Manager

> **Meta:** v1.0.0 | 22-02-2026 | Автор: Universal Coding Agent

## Обзор

Этот документ описывает обновленный рабочий процесс для режима Project Manager, который теперь адаптируется к различным фазам разработки, координируясь с соответствующими специалистами. Режим больше не зависит исключительно от System Analyst, а вместо этого определяет текущую фазу проекта и работает с соответствующим специалистом.

## Фазы проекта и координация со специалистами

### 1. Фаза инициации проекта
**Основной специалист:** System Analyst

**Активности Project Manager:**
- Координация с System Analyst для сбора требований
- Сбор пользовательских историй и сценариев использования
- Получение функциональных и нефункциональных требований
- Документирование технических ограничений
- Расчет начальных оценок на основе анализа требований

**Ключевые результаты:**
- Устав проекта
- Документация требований
- Начальные оценки проекта
- Оценка рисков на основе требований

### 2. Фаза проектирования
**Основной специалист:** Architect mode

**Активности Project Manager:**
- Координация с Architect mode для принятия технических решений
- Сбор архитектурных чертежей и диаграмм
- Сбор решений по технологическому стеку
- Получение шаблонов интеграции и архитектуры безопасности
- Обновление графика проекта на основе архитектурных решений

**Ключевые результаты:**
- Документация технической архитектуры
- Отчет о выборе технологий
- План интеграции
- Оценка соответствия безопасности

### 3. Фаза разработки
**Основной специалист:** Code mode

**Активности Project Manager:**
- Координация с Code mode для отслеживания реализации
- Мониторинг прогресса разработки и скорости выполнения
- Отслеживание метрик качества кода
- Решение проблем и блокировок в разработке
- Управление распределением ресурсов для задач разработки

**Ключевые результаты:**
- Отчеты о прогрессе разработки
- Метрики качества кода
- Отчеты об использовании ресурсов
- Оценки вех разработки

### 4. Фаза тестирования
**Основной специалист:** Code/Test Engineer mode

**Активности Project Manager:**
- Координация с Code/Test Engineer для обеспечения качества
- Мониторинг метрик покрытия тестами
- Отслеживание отчетов об ошибках и их устранения
- Проведение бенчмаркинга производительности
- Обеспечение соответствия качеству перед релизом

**Ключевые результаты:**
- Отчеты о покрытии тестами
- Резюме отслеживания ошибок
- Результаты бенчмаркинга производительности
- Оценки ворот качества

### 5. Фаза развертывания
**Основной специалист:** DevOps/Code mode

**Активности Project Manager:**
- Координация с DevOps/Code mode для управления релизами
- Проверка готовности к развертыванию
- Мониторинг состояния инфраструктуры
- Подготовка процедур отката
- Установка мониторинга производства

**Ключевые результаты:**
- Оценка готовности к развертыванию
- Отчет о состоянии инфраструктуры
- Документация процедур отката
- Настройка мониторинга производства

### 6. Фаза сопровождения
**Основной специалист:** Различные специалисты по мере необходимости

**Активности Project Manager:**
- Координация с соответствующими специалистами для обновлений
- Мониторинг производственных проблем
- Планирование окон обслуживания
- Планирование будущих улучшений
- Управление техническим долгом

**Ключевые результаты:**
- Отчеты об отслеживании проблем
- Планирование обслуживания
- Планирование улучшений
- Оценка технического долга

## Поток решений по выбору специалиста

1. **Определение текущей фазы:** Определить, в какой фазе разработки находится проект
2. **Выбор соответствующего специалиста:** На основе фазы выбрать соответствующего специалиста
3. **Запрос соответствующих данных:** Запросить данные, специфичные для текущей фазы и специалиста
4. **Интеграция информации:** Объединить информацию от соответствующих специалистов для деятельности по управлению проектом
5. **Обновление статуса фазы:** Отслеживать переходы между фазами и корректировать координацию соответствующим образом

## Примеры запросов данных по фазам

### Запросы данных на фазе инициации
- "System Analyst, пожалуйста, предоставьте пользовательские истории и критерии приемки"
- "System Analyst, какие функциональные требования для этой функции?"
- "System Analyst, какие технические ограничения следует учитывать?"

### Запросы данных на фазе проектирования
- "Architect mode, пожалуйста, предоставьте чертеж технической архитектуры"
- "Architect mode, какой технологический стек вы рекомендуете для масштабируемости?"
- "Architect mode, какие требования к архитектуре безопасности?"

### Запросы данных на фазе разработки
- "Code mode, какой текущий прогресс по задачам разработки?"
- "Code mode, какие метрики качества кода для этого спринта?"
- "Code mode, какие проблемы в разработке блокируют прогресс?"

### Запросы данных на фазе тестирования
- "Code/Test Engineer, какой текущий процент покрытия тестами?"
- "Code/Test Engineer, какие ошибки не устранены и какова их серьезность?"
- "Code/Test Engineer, какие результаты бенчмаркинга производительности?"

## Переход между фазами

Project Manager должен обеспечивать плавные переходы между фазами, путем:

1. **Проверки:** Обеспечения завершения всех результатов текущей фазы
2. **Утверждения:** Получения необходимых одобрений перед переходом
3. **Коммуникации:** Информирования всех заинтересованных сторон о переходе фазы
4. **Координации:** Настройки координации с новым специалистом фазы
5. **Документирования:** Обновления проектной документации для отражения изменения фазы

## Отчетность по фазам

Отчеты должны четко указывать:
- Текущую фазу проекта
- Специалистов, участвующих в генерации данных
- Охватываемый временной период
- Любые произошедшие переходы фаз
- Метрики, относящиеся к текущей фазе

## Преимущества координации по фазам

1. **Повышенная точность:** Получение правильной информации от правильного специалиста
2. **Лучшая эффективность:** Снижение узких мест за счет работы с соответствующими экспертами
3. **Повышенное качество:** Использование знаний специалистов для принятия лучших решений
4. **Четкая ответственность:** Определение обязанностей по фазам и ролям
5. **Адаптивное управление:** Корректировка подхода к управлению в зависимости от потребностей проекта
