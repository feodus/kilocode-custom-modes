# Organizational Change Management

> **Meta:** v2.0.0 | 09-03-2026
> **Переименовано из:** change-management

## Назначение

Управление организационными изменениями — это системный подход к подготовке, внедрению и закреплению изменений на уровне организации. В отличие от управления запросами на изменения в проекте (change-request), этот навык фокусируется на организационной трансформации: внедрении новых процессов, реструктуризации, изменении корпоративной культуры и адаптации сотрудников к новым условиям работы.

Применяется для успешного проведения организационных изменений с минимизацией сопротивления и максимизацией принятия изменений персоналом.

## Когда использовать

Используйте этот навык:
- При внедрении новых процессов или методологий разработки
- При реструктуризации команды или подразделения
- При переходе на новые технологии или инструменты
- При слиянии или поглощении компаний
- При изменении корпоративной культуры
- При масштабных проектах цифровой трансформации
- При необходимости управления сопротивлением изменениям

**Важно:** Для управления изменениями в рамках проекта (change requests, impact assessment) используйте навык [`change-request`](change-request/SKILL.md).

## Модели управления изменениями

### ADKAR Model

Модель ADKAR (ProSci) включает пять последовательных этапов:

| Этап | Описание | Действия |
|------|----------|----------|
| **Awareness** (Осведомлённость) | Создание понимания необходимости изменений | Коммуникации, встречи, демонстрация проблем |
| **Desire** (Желание) | Формирование желания участвовать и поддерживать изменения | Мотивация, демонстрация выгод, работа с лидерами |
| **Knowledge** (Знания) | Предоставление знаний о том, как осуществлять изменения | Обучение, тренинги, документация |
| **Ability** (Способность) | Развитие навыков и способностей для применения знаний | Практика, наставничество, поддержка |
| **Reinforcement** (Закрепление) | Поддержание изменений через поощрение и признание | Метрики, обратная связь, награды |

### Kotter's 8-Step Model

Модель Kotter включает восемь шагов:

1. **Создание срочности** — демонстрация необходимости изменений
2. **Формирование направляющей коалиции** — создание команды лидеров изменений
3. **Формирование видения и стратегии** — определение будущего состояния
4. **Коммуникация видения** — распространение информации о changes
5. **Расширение прав и действий** — устранение препятствий, наделение полномочиями
6. **Создание краткосрочных побед** — достижение видимых результатов
7. **Консолидация изменений** — укрепление достигнутого
8. **Закрепление в культуре** — интеграция изменений в нормативную базу

## Функции

### 1. Анализ текущего состояния

Оценка организационной готовности к изменениям:
- Анализ текущих процессов и практик
- Оценка культурного контекста
- Выявление стейкхолдеров и их влияния
- Определение потенциальных точек сопротивления
- Оценка ресурсов и возможностей

### 2. Планирование изменения

Разработка стратегии управления изменениями:
- Определениеscope и целей изменения
- Выбор подходящей модели управления (ADKAR, Kotter)
- Разработка плана коммуникаций
- Планирование обучения и развития
- Определение метрик успеха

### 3. Управление стейкхолдерами

Эффективное взаимодействие с заинтересованными сторонами:
- Идентификация ключевых стейкхолдеров
- Анализ влияния и интересов
- Разработка стратегии взаимодействия
- Регулярные коммуникации
- Работа с возражениями

### 4. Управление сопротивлением

Преодоление сопротивления изменениям:
- Выявление источников сопротивления
- Применение техник работы с сопротивлением
- Вовлечение лидеров мнений
- Создание безопасного пространства для обратной связи
- Поэтапное внедрение

### 5. Коммуникация изменений

Эффективное информирование о изменениях:
- Разработка коммуникационной стратегии
- Создание ключевых сообщений
- Выбор каналов коммуникации
- Регулярные обновления
- Обратная связь и двусторонняя коммуникация

### 6. Обучение и развитие

Подготовка персонала к новым условиям:
- Определение потребностей в обучении
- Разработка программ обучения
- Проведение тренингов
- Создание материалов и документации
- Наставничество и поддержка

### 7. Мониторинг и корректировка

Отслеживание прогресса и внесение корректировок:
- Отслеживание метрик по каждому этапу ADKAR
- Регулярные опросы и оценки
- Анализ обратной связи
- Корректировка плана
- Управление рисками изменения

## Примеры использования

### Пример 1: Внедрение Agile-методологии

Ситуация: Компания переходит на Agile-подход разработки.

План по ADKAR:
- **Осведомлённость:** Презентации о преимуществах Agile, демонстрация проблем текущего подхода
- **Желание:** Вовлечение команды в пилотирование, демонстрация ранних успехов
- **Знания:** Тренинги по Scrum/Kanban, Pair programming, обучение инструментам
- **Способность:** Наставничество, ретроспективы, постоянная поддержка
- **Закрепление:** Метрики команды, recognition, интеграция в绩效评估

Метрики успеха:
- Velocity стабилизировался
- Sprint burndown соблюдается
- Удовлетворённость команды > 4/5
- Time-to-market сократился на 30%

### Пример 2: Внедрение новой CRM-системы

План внедрения:
1. Формирование коалиции: Sponsors (руководство), Champions (лидеры мнений), Agents (внутренние тренеры)
2. Коммуникации: Town halls, emails, FAQ, intranet portal
3. Обучение: Tiered approach (администраторы → power users → end users)
4. Поддержка: Help desk, champions network, knowledge base

Управление сопротивлением:
- Работа с лидерами мнений на ранних этапах
- Демонстрация выгод для каждой роли
- Постепенный переход (phased rollout)
- Быстрые wins и успехи

### Пример 3: Реструктуризация команды

Ситуация: Объединение двух отделов разработки.

Этапы:
1. Анализ: Аудит обеих команд, выявление культурных различий
2. Планирование: Определение новой структуры, ролей, процессов
3. Коммуникация: Прозрачное информирование о причинах и планах
4. Интеграция: Совместные activities, team building
5. Закрепление: Новые нормы, ценности, процессы

Риски и митигация:
- Сопротивление → Работа с лидерами, прозрачность
- Потеря ключевых сотрудников → Retention bonuses, career paths
- Культурный конфликт → Формирование общих ценностей

## Метрики организационных изменений

### ADKAR Metrics

| Этап | Метрика | Целевое значение |
|------|---------|------------------|
| Awareness | % осведомлённых о необходимости изменений | > 90% |
| Desire | % желающих поддержать изменения | > 75% |
| Knowledge | % прошедших обучение | 100% |
| Ability | % способных применить знания | > 85% |
| Reinforcement | % закрепивших изменения | > 80% |

### Организационные метрики

- Employee engagement score
- Adoption rate новых процессов
- Time to productivity
- Change resistance incidents
- Stakeholder satisfaction

## Частые ошибки

1. **Пропуск этапов ADKAR** — изменение не закрепляется
2. **Недостаточная коммуникация** — слухи и недопонимание
3. **Игнорирование сопротивления** — конфликты и провал изменений
4. **Отсутствие быстрых побед** — потеря momentum
5. **Недостаточное закрепление** — возврат к старому
