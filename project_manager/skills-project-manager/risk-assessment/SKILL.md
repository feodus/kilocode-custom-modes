# Risk Assessment

> **Meta:** v1.0.0 | 22-02-2026

## Назначение

Навык для идентификации, анализа и оценки рисков проекта. Включает создание матриц рисков, определение вероятности и влияния рисков, разработку стратегий митигации и планов реагирования на риски.

## Когда использовать

Используйте этот навык:
- На этапе планирования проекта для первоначальной оценки рисков
- При возникновении новых рисков в процессе выполнения проекта
- Для регулярного обзора и обновления реестра рисков
- При необходимости определить влияние рисков на сроки и бюджет

## Функции

### Risk Identification
Идентификация потенциальных рисков:
- Технические риски
- Организационные риски
- Финансовые риски
- Риски, связанные с ресурсами
- Внешние риски

### Risk Analysis
Анализ выявленных рисков:
- Оценка вероятности возникновения
- Оценка потенциального влияния
- Определение категории риска
- Приоритизация рисков
- Анализ триггеров рисков

### Risk Matrix Creation
Создание матрицы рисков:
- Определение шкал оценки
- Визуализация рисков
- Классификация по уровням
- Определение зон ответственности
- Установка порогов для мониторинга

### Mitigation Strategies
Разработка стратегий митигации:
- Избегание рисков
- Передача рисков (аутсорсинг)
- Снижение вероятности/воздействия
- Планирование реагирования
- Резервирование ресурсов

## Интеграция с System Analyst

- Получение информации о технических рисках от System Analyst
- Анализ архитектурных рисков
- Учет рекомендаций по технологиям
- Оценка влияния технических решений на риски
- Валидация стратегий митигации с технической точки зрения

## Примеры использования

### Пример 1: Оценка рисков для проекта разработки веб-приложения
Технические риски:
- Риск: Недостаточная квалификация команды в новых технологиях
  - Вероятность: Средняя
  - Влияние: Высокое
  - Стратегия: Обучение команды, найм специалистов

- Риск: Проблемы с интеграцией с внешними API
  - Вероятность: Высокая
  - Влияние: Среднее
  - Стратегия: Раннее прототипирование, резерв времени

Организационные риски:
- Риск: Изменение требований в процессе разработки
  - Вероятность: Высокая
  - Влияние: Высокое
  - Стратегия: Четкий процесс управления изменениями

### Пример 2: Оценка рисков для проекта внедрения ERP-системы
Финансовые риски:
- Риск: Превышение бюджета из-за сложности интеграции
  - Вероятность: Средняя
  - Влияние: Высокое
  - Стратегия: Резерв бюджета 15%, поэтапное внедрение

Внешние риски:
- Риск: Изменение законодательства в процессе проекта
  - Вероятность: Низкая
  - Влияние: Высокое
  - Стратегия: Мониторинг изменений, юридическая экспертиза
