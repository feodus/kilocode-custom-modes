# Development Tracking

> **Meta:** v1.0.0 | 22-02-2026

## Purpose

Skill for tracking software development progress. Includes monitoring task completion, analyzing burn-down charts, measuring team velocity, forecasting delivery dates, and managing plan deviations. Helps maintain development process transparency and make informed management decisions.

## When to Use

Use this skill:
- When regularly monitoring project progress
- When forecasting completion dates is needed
- When analyzing team performance
- When managing schedule deviations
- When preparing reports for stakeholders

## Functions

### Progress Monitoring
Monitoring progress:
- Tracking task completion
- Analyzing backlog status
- Monitoring sprints (in Agile)
- Identifying plan deviations
- Identifying problem areas

### Burn-down Chart Analysis
Analyzing burn-down charts:
- Building completion charts
- Analyzing completion trends
- Forecasting work completion
- Identifying schedule deviations
- Assessing timeline risks

### Velocity Tracking
Tracking velocity:
- Measuring team performance
- Analyzing historical data
- Forecasting future velocity
- Comparing with benchmarks
- Accounting for team changes

### Delivery Forecasting
Forecasting delivery:
- Calculating remaining work volume
- Forecasting completion dates
- Analyzing change impact on timelines
- Building completion scenarios
- Accounting for risks in forecasts

## Integration with System Analyst

- Accounting for analytical tasks in tracking
- Monitoring progress on analytical work
- Tracking architectural task completion
- Accounting for requirements analysis time
- Integrating analytical results into backlog

## Usage Examples

### Example 1: Tracking Progress in Agile Project
Burn-down chart:
- Initially: 100 story points
- Remaining after 2 weeks: 70 story points
- Trend shows stable completion
- Forecast: completion in 6 weeks

Velocity:
- Average velocity: 15 story points per sprint
- Variations: from 12 to 18 story points
- Forecast for next sprint: 14-16 story points

Forecasting:
- 50 story points remaining
- At current velocity: 3-4 sprints
- Expected completion: in 6-8 weeks

### Example 2: Tracking Progress in Waterfall Project
Progress monitoring:
- Phase 1 (Design): 100% completed
- Phase 2 (Development): 60% completed
- Phase 3 (Testing): 0% started
- Overall progress: 45% completed

Timeline tracking:
- Project start: 01.01.2026
- Current date: 01.03.2026
- Planned completion date: 01.07.2026
- Actual progress: on schedule
- Risks: external API delay by 2 weeks
