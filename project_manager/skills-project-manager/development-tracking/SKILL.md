# Development Tracking

> **Meta:** v1.0.0 | 22-02-2026

## Назначение

Навык для отслеживания прогресса разработки программного обеспечения. Включает мониторинг выполнения задач, анализ бёрндаун-диаграмм, измерение велосити команды, прогнозирование сроков доставки и управление отклонениями от плана. Помогает поддерживать прозрачность процесса разработки и принимать обоснованные управленческие решения.

## Когда использовать

Используйте этот навык:
- При регулярном мониторинге прогресса проекта
- При необходимости прогнозирования сроков завершения
- При анализе производительности команды
- При управлении отклонениями от графика
- При подготовке отчетов для стейкхолдеров

## Функции

### Progress Monitoring
Мониторинг прогресса:
- Отслеживание выполнения задач
- Анализ статуса бэклога
- Мониторинг спринтов (в Agile)
- Выявление отклонений от плана
- Идентификация проблемных зон

### Burn-down Chart Analysis
Анализ бёрндаун-диаграмм:
- Построение диаграмм выполнения
- Анализ трендов выполнения
- Прогнозирование завершения работ
- Выявление отклонений от графика
- Оценка рисков по срокам

### Velocity Tracking
Отслеживание велосити:
- Измерение производительности команды
- Анализ исторических данных
- Прогнозирование будущей велосити
- Сравнение с эталонными значениями
- Учет изменений в команде

### Delivery Forecasting
Прогнозирование доставки:
- Расчет оставшегося объема работ
- Прогнозирование сроков завершения
- Анализ влияния изменений на сроки
- Построение сценариев завершения
- Учет рисков в прогнозах

## Интеграция с System Analyst

- Учет аналитических задач в отслеживании
- Мониторинг прогресса по аналитическим работам
- Отслеживание выполнения архитектурных задач
- Учет времени на анализ требований
- Интеграция аналитических результатов в бэклог

## Примеры использования

### Пример 1: Отслеживание прогресса в Agile-проекте
Burn-down диаграмма:
- Изначально: 100 story points
- Осталось через 2 недели: 70 story points
- Тренд показывает стабильное выполнение
- Прогноз: завершение за 6 недель

Velocity:
- Средняя велосити: 15 story points за спринт
- Вариации: от 12 до 18 story points
- Прогноз на следующий спринт: 14-16 story points

Прогнозирование:
- Осталось 50 story points
- При текущей велосити: 3-4 спринта
- Ожидаемое завершение: через 6-8 недель

### Пример 2: Отслеживание прогресса в Waterfall-проекте
Progress monitoring:
- Phase 1 (Design): 100% completed
- Phase 2 (Development): 60% completed
- Phase 3 (Testing): 0% started
- Overall progress: 45% completed

Timeline tracking:
- Начало проекта: 01.01.2026
- Текущая дата: 01.03.2026
- Плановая дата завершения: 01.07.2026
- Фактический прогресс: по графику
- Риски: задержка внешнего API на 2 недели
