# Project Architecture Overview

> **Meta:** v1.0.0 | 22-02-2026

## Назначение

Навык для понимания и управления архитектурными аспектами проекта с точки зрения проектного менеджмента. Включает анализ компонентов системы, их взаимодействия, зависимостей и интеграций, что позволяет эффективно планировать и управлять проектом разработки.

## Когда использовать

Используйте этот навык:
- При планировании проекта для понимания сложности архитектуры
- При определении зависимостей между компонентами
- При управлении рисками, связанными с архитектурой
- При планировании интеграционных работ
- При координации работы команд, работающих над разными компонентами

## Функции

### Component Mapping
Анализ компонентов системы:
- Идентификация основных модулей
- Определение границ компонентов
- Понимание ответственностей компонентов
- Определение владельцев компонентов
- Анализ взаимодействия между компонентами

### Dependency Analysis
Анализ зависимостей:
- Внутренние зависимости между компонентами
- Внешние зависимости (библиотеки, сервисы)
- Зависимости от инфраструктуры
- Временные зависимости
- Технологические зависимости

### Integration Points
Определение точек интеграции:
- API и интерфейсы
- Протоколы взаимодействия
- Форматы данных
- Временные аспекты интеграции
- Требования к синхронизации

### Architecture Complexity Assessment
Оценка сложности архитектуры:
- Оценка когнитивной нагрузки
- Оценка рисков сложности
- Оценка масштабируемости
- Оценка поддержки и сопровождения
- Оценка рисков интеграции

## Интеграция с System Analyst

- Получение архитектурных решений от System Analyst
- Уточнение требований к интеграциям
- Анализ влияния архитектурных решений на проект
- Учет рекомендаций по технологиям
- Понимание архитектурных ограничений

## Примеры использования

### Пример 1: Архитектурный обзор для веб-приложения
Компоненты системы:
- Frontend (React): отвечает за пользовательский интерфейс
- Backend API (Node.js): обработка бизнес-логики
- База данных (PostgreSQL): хранение данных
- Аутентификационный сервис (JWT): управление доступом
- Файловое хранилище (AWS S3): хранение файлов

Зависимости:
- Frontend зависит от Backend API
- Backend зависит от БД и аутентификационного сервиса
- Аутентификационный сервис интегрируется с Backend

Интеграции:
- Frontend ↔ Backend: REST API
- Backend ↔ БД: SQL
- Backend ↔ Аутентификация: JWT
- Backend ↔ S3: AWS SDK

### Пример 2: Архитектурный обзор для микросервисного приложения
Компоненты системы:
- Service Discovery (Consul): регистрация и поиск сервисов
- API Gateway (Kong): маршрутизация запросов
- User Service: управление пользователями
- Order Service: управление заказами
- Payment Service: обработка платежей
- Notification Service: уведомления

Зависимости:
- API Gateway зависит от Service Discovery
- Order Service зависит от User Service и Payment Service
- Notification Service зависит от других сервисов

Интеграции:
- Синхронные вызовы через REST
- Асинхронные вызовы через Message Queue
- Событийно-ориентированная архитектура
