# Resource Planning

> **Meta:** v1.0.0 | 22-02-2026

## Purpose

Skill for planning project resources, including defining team composition, capacity estimation, creating skills matrix, and role distribution. Helps effectively manage human and material resources throughout the project.

## When to Use

Use this skill:
- At project initiation and planning stage
- When forming project team
- When needing to assess resource availability
- For managing team workload and preventing overloads
- When planning hiring or outsourcing

## Functions

### Team Composition Planning
Defining project team composition:
- Defining required roles
- Estimating number of specialists
- Defining qualification requirements
- Planning work schedules
- Accounting for internal and external resources

### Capacity Planning
Planning team capacity:
- Estimating available time of resources
- Accounting for vacations and sick leaves
- Estimating team effectiveness
- Planning workload
- Balancing load

### Skills Matrix Development
Creating skills matrix:
- Assessing current competencies
- Identifying skills gaps
- Planning training
- Estimating knowledge concentration risks
- Planning team development

### Resource Allocation
Allocating resources to tasks:
- Assigning performers
- Accounting for dependencies
- Optimizing workload
- Balancing team
- Accounting for task priorities

## Integration with System Analyst

- Receiving information about technical resource requirements from System Analyst
- Accounting for technology recommendations in skills planning
- Estimating need for specialists in specific technologies
- Planning resources for analytical tasks
- Accounting for System Analyst workload in project

## Usage Examples

### Example 1: Resource Planning for Web Application Development Project
Team composition:
- 1 Project Manager (100% rate)
- 1 System Analyst (50% rate)
- 2 Full-stack developers (100% rate each)
- 1 UI/UX designer (50% rate)
- 1 QA engineer (100% rate)

Team capacity:
- Total capacity: 5.5 FTE
- Accounting for vacations: 5.0 FTE
- Effectiveness: 85%
- Available capacity: ~4.25 FTE

### Example 2: Resource Planning for ERP Implementation Project
Team composition:
- 1 Project Manager (100%)
- 1 Business Analyst (100%)
- 2 System Analysts (100% each)
- 2 Developers (100% each)
- 1 Data Migration Specialist (100%)
- 1 Trainer (25%)

Skills matrix:
- ERP systems: high level at Business Analyst and System Analyst
- SQL: medium level at all technical specialists
- Integrations: high level at System Analyst
- User training: high level at Trainer
