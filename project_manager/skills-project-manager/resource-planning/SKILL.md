# Resource Planning

> **Meta:** v1.0.0 | 22-02-2026

## Назначение

Навык для планирования ресурсов проекта, включая определение состава команды, оценку ёмкости, создание матрицы навыков и распределение ролей. Помогает эффективно управлять человеческими и материальными ресурсами на протяжении всего проекта.

## Когда использовать

Используйте этот навык:
- На этапе инициации и планирования проекта
- При формировании команды проекта
- При необходимости оценить доступность ресурсов
- Для управления загрузкой команды и предотвращения перегрузок
- При планировании найма или аутсорсинга

## Функции

### Team Composition Planning
Определение состава команды проекта:
- Определение необходимых ролей
- Оценка количества специалистов
- Определение квалификационных требований
- Планирование графиков работы
- Учет внутренних и внешних ресурсов

### Capacity Planning
Планирование ёмкости команды:
- Оценка доступного времени ресурсов
- Учет отпусков и больничных
- Оценка эффективности команды
- Планирование загрузки
- Балансировка нагрузки

### Skills Matrix Development
Создание матрицы навыков:
- Оценка текущих компетенций
- Определение пробелов в навыках
- Планирование обучения
- Оценка рисков сосредоточения знаний
- Планирование развития команды

### Resource Allocation
Распределение ресурсов по задачам:
- Назначение исполнителей
- Учет зависимостей
- Оптимизация загрузки
- Балансировка команды
- Учет приоритетов задач

## Интеграция с System Analyst

- Получение информации о технических требованиях к ресурсам от System Analyst
- Учет рекомендаций по технологиям при планировании навыков
- Оценка необходимости специалистов в определенных технологиях
- Планирование ресурсов для аналитических задач
- Учет нагрузки System Analyst в проекте

## Примеры использования

### Пример 1: Планирование ресурсов для проекта разработки веб-приложения
Состав команды:
- 1 Project Manager (100% ставка)
- 1 System Analyst (50% ставка)
- 2 Full-stack разработчика (100% ставка каждый)
- 1 UI/UX дизайнер (50% ставка)
- 1 QA-инженер (100% ставка)

Ёмкость команды:
- Общая ёмкость: 5.5 FTE
- С учетом отпусков: 5.0 FTE
- Эффективность: 85%
- Доступная ёмкость: ~4.25 FTE

### Пример 2: Планирование ресурсов для проекта внедрения ERP-системы
Состав команды:
- 1 Project Manager (100%)
- 1 Business Analyst (100%)
- 2 System Analyst (100% каждый)
- 2 Developer (100% каждый)
- 1 Data Migration Specialist (100%)
- 1 Trainer (25%)

Матрица навыков:
- ERP-системы: высокий уровень у Business Analyst и System Analyst
- SQL: средний уровень у всех технических специалистов
- Интеграции: высокий уровень у System Analyst
- Обучение пользователей: высокий уровень у Trainer
