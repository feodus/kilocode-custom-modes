# Development Planning

> **Meta:** v1.0.0 | 22-02-2026

## Назначение

Навык для планирования фазы разработки программного обеспечения. Включает создание детального расписания разработки, определение вех, планирование результатов и управление переходами между фазами проекта. Помогает эффективно управлять процессом разработки и контролировать его выполнение.

## Когда использовать

Используйте этот навык:
- При планировании фазы разработки проекта
- При создании детального графика работ
- При определении ключевых вех и результатов
- При управлении зависимостями между задачами разработки
- При планировании ресурсов для фазы разработки

## Функции

### Development Schedule Creation
Создание расписания разработки:
- Определение фаз разработки
- Планирование итераций (в Agile)
- Учет времени на проектирование
- Планирование кодирования
- Учет времени на тестирование

### Milestone Planning
Планирование ключевых вех:
- Определение контрольных точек
- Установка дат вех
- Определение критериев достижения вех
- Планирование демонстраций
- Учет внешних зависимостей

### Deliverable Planning
Планирование результатов:
- Определение промежуточных результатов
- Планирование инкрементов (в Agile)
- Установка критериев приемки
- Планирование артефактов
- Учет требований к качеству

### Phase Transition Management
Управление переходами между фазами:
- Определение критериев перехода
- Планирование проверок качества
- Учет результатов предыдущих фаз
- Управление зависимостями
- Планирование ретроспектив

## Интеграция с System Analyst

- Получение архитектурных решений от System Analyst
- Учет технических требований при планировании
- Планирование задач анализа в фазе разработки
- Координация работ System Analyst с разработкой
- Учет рекомендаций по технологиям

## Примеры использования

### Пример 1: Планирование разработки в Agile-проекте
Фазы разработки:
- Sprint 1: Настройка окружения, базовая архитектура (2 недели)
- Sprint 2: Аутентификация и авторизация (2 недели)
- Sprint 3: Базовый функционал (2 недели)
- Sprint 4: Расширенный функционал (2 недели)
- Sprint 5: Тестирование и исправление (2 недели)

Вехи:
- Milestone 1: Рабочая архитектура (после Sprint 1)
- Milestone 2: Основной функционал (после Sprint 3)
- Milestone 3: Готовый продукт (после Sprint 5)

Результаты:
- Рабочий прототип после Sprint 1
- Функциональная система после Sprint 3
- Готовый продукт после Sprint 5

### Пример 2: Планирование разработки в Waterfall-проекте
Фазы разработки:
- Фаза 1: Детальное проектирование (4 недели)
- Фаза 2: Разработка Frontend (6 недель)
- Фаза 3: Разработка Backend (8 недель)
- Фаза 4: Интеграция (2 недели)
- Фаза 5: Тестирование (3 недели)

Вехи:
- Milestone 1: Детальный дизайн завершен (после Фазы 1)
- Milestone 2: Frontend завершен (после Фазы 2)
- Milestone 3: Backend завершен (после Фазы 3)
- Milestone 4: Интеграция завершена (после Фазы 4)
- Milestone 5: Продукт готов (после Фазы 5)

Критерии перехода:
- Переход к фазе разработки: утвержден дизайн
- Переход к интеграции: завершена разработка
- Переход к тестированию: завершена интеграция
