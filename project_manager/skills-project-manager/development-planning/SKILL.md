# Development Planning

> **Meta:** v1.0.0 | 22-02-2026

## Purpose

Skill for planning the software development phase. Includes creating detailed development schedule, defining milestones, planning deliverables, and managing transitions between project phases. Helps effectively manage the development process and control its execution.

## When to Use

Use this skill:
- When planning the project's development phase
- When creating a detailed work schedule
- When defining key milestones and deliverables
- When managing dependencies between development tasks
- When planning resources for the development phase

## Functions

### Development Schedule Creation
Creating development schedule:
- Defining development phases
- Planning iterations (in Agile)
- Accounting for design time
- Planning coding
- Accounting for testing time

### Milestone Planning
Planning key milestones:
- Defining checkpoints
- Setting milestone dates
- Defining milestone achievement criteria
- Planning demonstrations
- Accounting for external dependencies

### Deliverable Planning
Planning deliverables:
- Defining intermediate deliverables
- Planning increments (in Agile)
- Setting acceptance criteria
- Planning artifacts
- Accounting for quality requirements

### Phase Transition Management
Managing phase transitions:
- Defining transition criteria
- Planning quality gates
- Accounting for previous phase results
- Managing dependencies
- Planning retrospectives

## Integration with System Analyst

- Receiving architectural decisions from System Analyst
- Accounting for technical requirements in planning
- Planning analysis tasks in the development phase
- Coordinating System Analyst work with development
- Accounting for technology recommendations

## Usage Examples

### Example 1: Development Planning in Agile Project
Development phases:
- Sprint 1: Environment setup, basic architecture (2 weeks)
- Sprint 2: Authentication and authorization (2 weeks)
- Sprint 3: Basic functionality (2 weeks)
- Sprint 4: Extended functionality (2 weeks)
- Sprint 5: Testing and fixes (2 weeks)

Milestones:
- Milestone 1: Working architecture (after Sprint 1)
- Milestone 2: Core functionality (after Sprint 3)
- Milestone 3: Ready product (after Sprint 5)

Deliverables:
- Working prototype after Sprint 1
- Functional system after Sprint 3
- Ready product after Sprint 5

### Example 2: Development Planning in Waterfall Project
Development phases:
- Phase 1: Detailed design (4 weeks)
- Phase 2: Frontend development (6 weeks)
- Phase 3: Backend development (8 weeks)
- Phase 4: Integration (2 weeks)
- Phase 5: Testing (3 weeks)

Milestones:
- Milestone 1: Detailed design completed (after Phase 1)
- Milestone 2: Frontend completed (after Phase 2)
- Milestone 3: Backend completed (after Phase 3)
- Milestone 4: Integration completed (after Phase 4)
- Milestone 5: Product ready (after Phase 5)

Transition criteria:
- Transition to development phase: design approved
- Transition to integration: development completed
- Transition to testing: integration completed
