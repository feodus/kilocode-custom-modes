# Testing Coordination

> **Meta:** v1.0.0 | 22-02-2026

## Назначение

Навык для координации процесса тестирования программного обеспечения. Включает планирование тестирования, подготовку тестовой среды, отслеживание дефектов, определение критериев релиза и обеспечение качества продукта. Помогает организовать эффективный процесс тестирования и гарантировать, что продукт соответствует требованиям.

## Когда использовать

Используйте этот навык:
- При планировании фазы тестирования проекта
- При подготовке тестовой инфраструктуры
- При координации работы команды тестирования
- При управлении процессом выявления и исправления дефектов
- При определении готовности продукта к релизу

## Функции

### Test Planning Coordination
Координация планирования тестирования:
- Определение уровней тестирования
- Планирование типов тестов
- Установка сроков тестирования
- Распределение ресурсов на тестирование
- Определение критериев начала тестирования

### Test Environment Setup
Подготовка тестовой среды:
- Координация настройки окружений
- Обеспечение доступа к средам
- Поддержание актуальности данных
- Обеспечение изолированности сред
- Планирование обновлений сред

### Defect Tracking Management
Управление отслеживанием дефектов:
- Настройка процесса регистрации
- Классификация и приоритизация дефектов
- Отслеживание статуса исправления
- Анализ трендов дефектов
- Управление SLA по исправлению

### Release Criteria Definition
Определение критериев релиза:
- Определение порогов качества
- Установка критериев прохождения тестов
- Проверка соответствия требованиям
- Оценка рисков оставшихся дефектов
- Утверждение готовности к релизу

## Интеграция с System Analyst

- Учет нефункциональных требований при планировании тестирования
- Использование сценариев от System Analyst для тестирования
- Проверка соответствия требованиям
- Участие System Analyst в анализе дефектов
- Обновление требований на основе результатов тестирования

## Примеры использования

### Пример 1: Координация тестирования веб-приложения
Типы тестирования:
- Unit тесты: 100% покрытие критических путей
- Integration тесты: проверка интеграции с внешними API
- E2E тесты: ключевые пользовательские сценарии
- Performance тесты: нагрузка до 1000 одновременных пользователей
- Security тесты: проверка уязвимостей OWASP Top 10

Тестовые среды:
- Development: для разработчиков
- Staging: для интеграционного тестирования
- Pre-production: для приемочного тестирования
- Production: для UAT и нагрузочного тестирования

Критерии релиза:
- 0 критических и 0 высоких дефектов
- 95% успешных результатов автоматических тестов
- Время отклика не более 2 секунд при нагрузке
- Пройдено не менее 90% тест-кейсов

### Пример 2: Управление дефектами в проекте
Процесс:
- Регистрация: через JIRA
- Классификация: по типу и приоритету
- Назначение: ответственным разработчикам
- SLA: критические - 24 часа, высокие - 3 дня, средние - 1 неделя
- Статусы: Open → In Progress → Resolved → Closed

Отслеживание:
- Ежедневные обзоры статуса дефектов
- Недельные отчеты по трендам
- Метрики: плотность дефектов, время исправления
- Ретроспективы: анализ причин дефектов

### Пример 3: Планирование приемочного тестирования
UAT планирование:
- Участники: ключевые пользователи и бизнес-аналитики
- Сценарии: на основе пользовательских историй
- Инфраструктура: pre-production среда
- Длительность: 2 недели
- Критерии успеха: 95% сценариев выполнены успешно

Координация:
- Подготовка тестовых данных
- Обучение пользователей процессу тестирования
- Сбор и анализ обратной связи
- Итеративное улучшение продукта
- Подготовка заключения о готовности
