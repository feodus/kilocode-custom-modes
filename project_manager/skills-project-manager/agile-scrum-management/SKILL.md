# Agile Scrum Management

> **Meta:** v1.0.0 | 22-02-2026

## Purpose

Skill for managing projects using Scrum methodology. Includes planning and conducting sprints, managing product and sprint backlogs, conducting Scrum ceremonies, tracking progress, and ensuring transparency in the development process. Helps the team work effectively in conditions of uncertainty and frequent requirement changes.

## When to Use

Use this skill:
- When managing projects using Scrum methodology
- When product requirements change frequently
- When frequent value delivery to the customer is needed
- When the team is ready for self-organization and collaboration
- When the customer actively participates in the development process

## Functions

### Sprint Planning
Sprint planning:
- Defining sprint goals
- Selecting items from the product backlog
- Estimating effort for tasks
- Planning work for the sprint
- Detailing user stories

### Daily Standup Facilitation
Conducting daily standups:
- Organizing the meeting
- Collecting task completion status
- Identifying blockers
- Facilitating team communication
- Ensuring process transparency

### Sprint Review and Retrospective
Conducting reviews and retrospectives:
- Demonstrating sprint results
- Collecting feedback from stakeholders
- Analyzing achieved results
- Identifying process improvements
- Adapting the approach to work

### Backlog Management
Backlog management:
- Prioritizing tasks
- Detailing user stories
- Estimating task complexity
- Incorporating feedback
- Keeping the backlog up-to-date

## Integration with System Analyst

- Receiving user stories from System Analyst
- Clarifying requirements during sprints
- Involving System Analyst in ceremonies when needed
- Considering architectural decisions in planning
- Planning analytical tasks in the backlog

## Usage Examples

### Example 1: Web Application Sprint Management
Sprint Planning:
- Sprint goal: "Implement authentication module"
- Tasks:
  - Create registration form
  - Implement data validation
  - Set up authentication API integration
  - Write unit tests
- Estimation: 30 story points

Daily Standup:
- What I did yesterday: completed form validation
- What I'll do today: start API integration
- Any blockers: no

Sprint Review:
- Demonstration: registration and login form
- Feedback: additional validation needed

Sprint Retrospective:
- What went well: good team communication
- What to improve: speed up code review process

### Example 2: Product Backlog Management
Backlog Refinement:
- User Story: "As a user, I want the ability to reset my password to regain access to my account"
- Acceptance Criteria:
  - Password reset request form
  - Email with reset link
  - New password setup page
- Estimation: 8 story points
- Priority: High
