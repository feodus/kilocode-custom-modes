# Agile Scrum Management

> **Meta:** v1.0.0 | 22-02-2026

## Назначение

Навык для управления проектами по Scrum-методологии. Включает планирование и проведение спринтов, управление бэклогом продукта и спринта, проведение Scrum-церемоний, отслеживание прогресса и обеспечение прозрачности процесса разработки. Помогает команде эффективно работать в условиях неопределенности и частых изменений требований.

## Когда использовать

Используйте этот навык:
- При управлении проектами по Scrum-методологии
- Когда требования к продукту часто изменяются
- При необходимости частой доставки ценности заказчику
- Когда команда готова к самоорганизации и сотрудничеству
- При активном участии заказчика в процессе разработки

## Функции

### Sprint Planning
Планирование спринтов:
- Определение целей спринта
- Выбор задач из бэклога продукта
- Оценка трудозатрат на задачи
- Планирование работ на спринт
- Детализация пользовательских историй

### Daily Standup Facilitation
Проведение ежедневных стендапов:
- Организация встречи
- Сбор статуса выполнения задач
- Выявление блокеров
- Содействие коммуникации команды
- Обеспечение прозрачности процесса

### Sprint Review and Retrospective
Проведение ревью и ретроспектив:
- Демонстрация результата спринта
- Сбор обратной связи от стейкхолдеров
- Анализ достигнутого результата
- Определение улучшений процесса
- Адаптация подхода к работе

### Backlog Management
Управление бэклогом:
- Приоритизация задач
- Детализация пользовательских историй
- Оценка сложности задач
- Учет обратной связи
- Поддержание актуальности бэклога

## Интеграция с System Analyst

- Получение пользовательских историй от System Analyst
- Уточнение требований в процессе спринтов
- Участие System Analyst в церемониях при необходимости
- Учет архитектурных решений при планировании
- Планирование аналитических задач в бэклоге

## Примеры использования

### Пример 1: Управление спринтом веб-приложения
Sprint Planning:
- Цель спринта: "Реализовать модуль аутентификации"
- Задачи:
  - Создать форму регистрации
  - Реализовать валидацию данных
  - Настроить интеграцию с API аутентификации
  - Написать unit-тесты
- Оценка: 30 story points

Daily Standup:
- Что сделал вчера: завершил валидацию формы
- Что буду делать сегодня: начну интеграцию с API
- Есть ли блокеры: нет

Sprint Review:
- Демонстрация: форма регистрации и входа
- Обратная связь: необходима дополнительная валидация

Sprint Retrospective:
- Что хорошо: хорошая коммуникация в команде
- Что улучшить: ускорить процесс code review

### Пример 2: Управление бэклогом продукта
Backlog Refinement:
- User Story: "Как пользователь, я хочу возможность сброса пароля, чтобы восстановить доступ к аккаунту"
- Acceptance Criteria:
  - Форма запроса сброса пароля
  - Отправка письма с ссылкой для сброса
  - Страница установки нового пароля
- Estimation: 8 story points
- Priority: High
