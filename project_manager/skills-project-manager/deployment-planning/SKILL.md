# Deployment Planning

> **Meta:** v1.0.0 | 22-02-2026

## Назначение

Навык для планирования процесса внедрения программного обеспечения в эксплуатацию. Включает разработку стратегии развертывания, подготовку плана отката, настройку целевой инфраструктуры, координацию запуска и управление рисками внедрения. Помогает обеспечить безопасное и надежное развертывание продукта в продакшн-окружение.

## Когда использовать

Используйте этот навык:
- При подготовке к запуску продукта в продакшн
- При планировании стратегии развертывания
- При необходимости подготовки плана отката
- При координации действий команд во время внедрения
- При управлении рисками процесса внедрения

## Функции

### Deployment Strategy Design
Разработка стратегии развертывания:
- Определение подхода к развертыванию
- Выбор метода (канареечный, сине-зеленый, поэтапный)
- Планирование времени внедрения
- Учет требований к доступности
- Определение окончаний технического обслуживания

### Rollback Plan Preparation
Подготовка плана отката:
- Определение триггеров для отката
- Планирование шагов отката
- Подготовка резервных копий
- Тестирование плана отката
- Определение ответственных за откат

### Environment Preparation
Подготовка целевой инфраструктуры:
- Проверка соответствия требованиям
- Настройка конфигураций
- Подготовка данных для миграции
- Проверка интеграций
- Тестирование инфраструктуры

### Go-Live Coordination
Координация запуска:
- Организация команды внедрения
- Управление коммуникациями
- Мониторинг процесса внедрения
- Управление инцидентами
- Поддержка после внедрения

## Интеграция с System Analyst

- Учет архитектурных решений при планировании развертывания
- Учет требований к инфраструктуре от System Analyst
- Планирование миграции данных по рекомендациям System Analyst
- Учет интеграционных требований
- Участие System Analyst в планировании тестирования

## Примеры использования

### Пример 1: Планирование внедрения веб-приложения
Стратегия:
- Canary deployment: 5% пользователей, затем 25%, затем 100%
- Время внедрения: 02:00 - 04:00 (минимальная нагрузка)
- Длительность: 4 часа (с учетом тестирования)

План отката:
- Триггеры: 50% ошибок в API, время отклика > 10с
- Время отката: до 30 минут
- Шаги: откат Docker-образа, восстановление БД
- Ответственный: DevOps-инженер

Инфраструктура:
- Load balancer: настройка весов
- База данных: резервная копия перед внедрением
- Мониторинг: Grafana, Alertmanager
- Логирование: ELK стек

### Пример 2: Подготовка окружения
Требования:
- CPU: 8 cores, RAM: 32GB
- Storage: 500GB SSD
- Network: 1Gbps
- OS: Ubuntu 20.04 LTS

Проверки:
- [ ] Сеть: доступность внешних API
- [ ] БД: подключение, производительность
- [ ] Безопасность: SSL-сертификаты
- [ ] Мониторинг: алерты и дашборды
- [ ] Резервное копирование: настройка и тестирование

### Пример 3: Команда внедрения
Состав:
- Project Manager: координация
- DevOps Engineer: техническая реализация
- System Analyst: проверка требований
- QA Lead: мониторинг качества
- Security Officer: проверка безопасности

Коммуникации:
- Slack-канал: #deployment-monitoring
- Ежечасные обновления статуса
- 24/7 поддержка в первые 48 часов
- Ретроспектива: через 1 неделю
