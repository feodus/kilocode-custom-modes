# UAT Coordination

> **Meta:** v1.0.0 | 22-02-2026

## Назначение

Навык для координации процесса приемочного тестирования (UAT - User Acceptance Testing). Включает планирование UAT, обучение пользователей, сбор и анализ обратной связи, управление процессом одобрения результата. Помогает обеспечить, что продукт соответствует ожиданиям конечных пользователей и бизнес-требованиям.

## Когда использовать

Используйте этот навык:
- При подготовке к приемочному тестированию
- При координации участия конечных пользователей
- При управлении процессом одобрения продукта
- При сборе и анализе обратной связи от пользователей
- При подготовке продукта к передаче в эксплуатацию

## Функции

### UAT Planning
Планирование приемочного тестирования:
- Определение участников UAT
- Планирование временных рамок
- Подготовка тестовых сценариев
- Определение критериев успеха
- Планирование ресурсов для поддержки

### User Training and Onboarding
Обучение пользователей:
- Подготовка обучающих материалов
- Проведение тренингов
- Подготовка документации для пользователей
- Организация сессий поддержки
- Создание каналов обратной связи

### Feedback Collection and Analysis
Сбор и анализ обратной связи:
- Настройка механизмов сбора
- Классификация и приоритизация замечаний
- Анализ тенденций и проблем
- Подготовка отчетов по результатам
- Определение требуемых изменений

### Sign-off Management
Управление процессом одобрения:
- Определение лиц с правом одобрения
- Подготовка чек-листов для одобрения
- Отслеживание статуса одобрения
- Управление процессом повторного тестирования
- Фиксация результатов и принятие решений

## Интеграция с System Analyst

- Использование пользовательских сценариев от System Analyst
- Учет требований, определенных System Analyst
- Вовлечение System Analyst в анализ обратной связи
- Участие System Analyst в сессиях обучения
- Интеграция результатов UAT в требования

## Примеры использования

### Пример 1: UAT для банковского веб-портала
Участники:
- Основные пользователи: клиенты банка (50 человек)
- Представители поддержки (5 человек)
- Бизнес-аналитики (3 человека)
- Представители регулятора (2 человека)

Тестовые сценарии:
- Открытие счета онлайн
- Перевод средств между счетами
- Оплата коммунальных услуг
- Просмотр выписки по счету
- Настройка уведомлений

Критерии успеха:
- 95% успешных сценариев
- Время выполнения не более 5 минут на сценарий
- Уровень удовлетворенности не менее 80%
- Наличие не более 5 низкоприоритетных замечаний

### Пример 2: Процесс сбора обратной связи
Механизмы:
- Онлайн анкеты после выполнения сценариев
- Фокус-группы (еженедельно)
- Сессии обратной связи (ежедневно)
- Трекер замечаний (JIRA)
- Рейтинг удовлетворенности (0-10)

Классификация:
- Critical: блокируют использование
- High: значительное влияние на UX
- Medium: улучшения удобства
- Low: косметические замечания

Анализ:
- Ежедневные отчеты по замечаниям
- Недельные тренд-анализы
- Сводка по категориям проблем
- Приоритизация для команды разработки

### Пример 3: Процесс одобрения
Sign-off чек-лист:
- [ ] Все критические сценарии работают корректно
- [ ] Пользовательские требования выполнены
- [ ] Производительность соответствует SLA
- [ ] Безопасность соответствует стандартам
- [ ] Документация предоставлена

Одобряющие:
- Product Owner: бизнес-функциональность
- Security Officer: аспекты безопасности
- Compliance Manager: соответствие нормативным требованиям
- Key End Users: удобство использования

Решение:
- Условия: не более 5 High-замечаний без решения
- Сроки: 3 дня на принятие решения
- Процесс: электронное одобрение с комментариями
