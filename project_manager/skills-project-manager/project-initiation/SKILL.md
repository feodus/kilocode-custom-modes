# Project Initiation

> **Meta:** v1.0.0 | 22-02-2026

## Назначение

Навык для инициации проекта, определения его целей, задач и основных параметров. Включает в себя создание устава проекта (Project Charter), бизнес-обоснования и идентификацию заинтересованных сторон.

## Когда использовать

Используйте этот навык:
- При запуске нового проекта
- Когда требуется формализовать цели и задачи проекта
- При необходимости создания устава проекта
- Для идентификации ключевых стейкхолдеров

## Функции

### Project Charter Creation
Создание устава проекта с ключевыми элементами:
- Цель проекта
- Область применения (scope)
- Основные результаты (deliverables)
- Высокоуровневые требования
- Ограничения и предположения
- Авторитет проектного менеджера
- Подпись спонсора

### Business Case Development
Разработка бизнес-обоснования:
- Обоснование проекта
- Ожидаемые выгоды
- Анализ заинтересованных сторон
- Оценка рисков
- Финансовое обоснование

### Stakeholder Identification
Идентификация ключевых заинтересованных сторон:
- Спонсоры проекта
- Клиенты
- Команда проекта
- Внешние поставщики
- Влияющие лица

### Goal and Objective Setting
Определение целей и задач проекта:
- Формулировка SMART-целей
- Приоритизация задач
- Определение критериев успеха
- Установка ожиданий

## Интеграция с System Analyst

- Получение бизнес-требований от System Analyst
- Анализ технических ограничений
- Учет архитектурных решений
- Использование данных о пользовательских потребностях

## Примеры использования

### Пример 1: Инициация проекта разработки веб-приложения
Цель: Создать веб-приложение для управления задачами
Область применения: Разработка и внедрение системы управления задачами для внутреннего использования
Основные результаты: Функциональное веб-приложение, документация, обучение пользователей
Критерии успеха: Запуск в течение 3 месяцев, удовлетворенность пользователей >80%

### Пример 2: Инициация проекта интеграции систем
Цель: Интегрировать CRM и ERP системы
Область применения: Разработка интерфейсов интеграции, настройка синхронизации данных
Основные результаты: Интеграционные интерфейсы, документация, протестированные процессы
Критерии успеха: Успешная синхронизация данных в реальном времени, минимизация downtime
