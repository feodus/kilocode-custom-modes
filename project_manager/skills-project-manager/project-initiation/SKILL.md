# Project Initiation

> **Meta:** v1.0.0 | 22-02-2026

## Purpose

Skill for project initiation, defining its goals, objectives, and main parameters. Includes creating Project Charter, business case, and stakeholder identification.

## When to Use

Use this skill:
- When launching a new project
- When formalizing project goals and objectives is needed
- When creating project charter is required
- For identifying key stakeholders

## Functions

### Project Charter Creation
Creating project charter with key elements:
- Project goal
- Scope
- Main deliverables
- High-level requirements
- Constraints and assumptions
- Project manager authority
- Sponsor signature

### Business Case Development
Developing business case:
- Project justification
- Expected benefits
- Stakeholder analysis
- Risk assessment
- Financial justification

### Stakeholder Identification
Identifying key stakeholders:
- Project sponsors
- Customers
- Project team
- External vendors
- Influencers

### Goal and Objective Setting
Defining project goals and objectives:
- Formulating SMART goals
- Prioritizing objectives
- Defining success criteria
- Setting expectations

## Integration with System Analyst

- Receiving business requirements from System Analyst
- Analyzing technical constraints
- Accounting for architectural decisions
- Using user needs data

## Usage Examples

### Example 1: Initiating Web Application Development Project
Goal: Create a web application for task management
Scope: Development and implementation of task management system for internal use
Main deliverables: Functional web application, documentation, user training
Success criteria: Launch within 3 months, user satisfaction >80%

### Example 2: Initiating System Integration Project
Goal: Integrate CRM and ERP systems
Scope: Developing integration interfaces, setting up data synchronization
Main deliverables: Integration interfaces, documentation, tested processes
Success criteria: Successful real-time data synchronization, minimized downtime
