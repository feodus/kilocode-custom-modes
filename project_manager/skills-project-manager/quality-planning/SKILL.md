# Quality Planning

> **Meta:** v1.0.0 | 22-02-2026

## Назначение

Навык для планирования качества программного обеспечения в фазе разработки. Включает определение контрольных точек качества, разработку стратегии тестирования, установление требований к соответствию стандартам и организация процессов обеспечения качества. Помогает гарантировать, что результаты проекта соответствуют установленным стандартам и требованиям.

## Когда использовать

Используйте этот навык:
- При планировании фазы разработки проекта
- При определении требований к качеству
- При создании стратегии тестирования
- При установлении контрольных точек качества
- При планировании процессов верификации и валидации

## Функции

### Quality Gate Definition
Определение контрольных точек качества:
- Установка точек проверки качества
- Определение критериев прохождения
- Планирование проверок артефактов
- Определение ответственных за проверки
- Установка процедур исправления дефектов

### Testing Strategy Development
Разработка стратегии тестирования:
- Планирование уровней тестирования
- Определение типов тестов
- Установка критериев завершения тестирования
- Планирование автоматизации тестирования
- Определение объема ручного тестирования

### Compliance Requirements Planning
Планирование соответствия требованиям:
- Определение стандартов качества
- Планирование соблюдения нормативных требований
- Установка процедур аудита
- Планирование процессов ревью
- Определение требований к документации

### Quality Assurance Process Design
Проектирование процессов обеспечения качества:
- Определение ролей и ответственностей
- Установка процедур контроля
- Планирование метрик качества
- Определение инструментов обеспечения качества
- Установка процессов непрерывного улучшения

## Интеграция с System Analyst

- Получение нефункциональных требований от System Analyst
- Учет требований к качеству при планировании
- Определение критериев качества на основе анализа
- Планирование тестирования архитектурных решений
- Учет рекомендаций по качеству от System Analyst

## Примеры использования

### Пример 1: Планирование качества для веб-приложения
Контрольные точки:
- Code Review Gate: перед слиянием изменений
- Unit Test Gate: покрытие не менее 80%
- Integration Test Gate: прохождение всех интеграционных тестов
- Security Scan Gate: отсутствие критических уязвимостей
- Performance Gate: время отклика не более 2 секунд

Стратегия тестирования:
- Unit тесты: разработчики, 100% покрытия критических путей
- Integration тесты: автоматизированные, покрытие всех API
- E2E тесты: автоматизированные, ключевые пользовательские сценарии
- Security тесты: автоматизированные сканеры и пентест
- Performance тесты: нагрузочные тесты на каждом релизе

### Пример 2: Планирование качества для банковского приложения
Контрольные точки:
- Architecture Review Gate: соответствие архитектурным стандартам
- Security Review Gate: прохождение аудита безопасности
- Compliance Gate: соответствие банковским стандартам
- Performance Gate: SLA по времени отклика
- Documentation Gate: полнота технической документации

Стратегия тестирования:
- Unit тесты: не менее 90% покрытия
- Contract тесты: проверка взаимодействия с внешними системами
- Security тесты: регулярные сканы и аудиты
- Compliance тесты: проверка соответствия банковским нормам
- Disaster Recovery тесты: проверка восстановления после сбоев
