# Quality Metrics

> **Meta:** v1.0.0 | 22-02-2026

## Назначение

Навык для определения, отслеживания и анализа метрик качества программного обеспечения. Включает измерение плотности дефектов, покрытия кода тестами, DORA-метрик (Lead Time, Deployment Frequency, MTTR, Change Failure Rate) и других показателей, характеризующих качество продукта и процессов разработки. Помогает принимать обоснованные решения по улучшению качества.

## Когда использовать

Используйте этот навык:
- При оценке текущего уровня качества
- При анализе эффективности процессов разработки
- При подготовке отчетов о качестве
- При принятии решений о готовности к релизу
- При планировании мероприятий по улучшению качества

## Функции

### Defect Density Measurement
Измерение плотности дефектов:
- Расчет количества дефектов на единицу кода
- Анализ трендов дефектов
- Сравнение с эталонными значениями
- Идентификация проблемных модулей
- Прогнозирование качества

### Code Coverage Analysis
Анализ покрытия кода тестами:
- Измерение процента покрытия
- Анализ покрытия критических путей
- Сравнение с целевыми показателями
- Идентификация непокрытых участков
- Оптимизация тестового покрытия

### DORA Metrics Tracking
Отслеживание DORA-метрик:
- Lead Time: время от коммита до деплоя
- Deployment Frequency: частота деплоев
- Mean Time To Recovery (MTTR): время восстановления
- Change Failure Rate: процент неудачных изменений
- Анализ влияния на бизнес-результаты

### Quality Dashboard Creation
Создание дашбордов качества:
- Визуализация ключевых метрик
- Настройка автоматических отчетов
- Интеграция данных из разных источников
- Настройка предупреждений
- Предоставление доступа стейкхолдерам

## Интеграция с System Analyst

- Учет качества аналитических артефактов
- Измерение точности аналитических требований
- Анализ влияния требований на качество кода
- Отслеживание метрик архитектурных решений
- Учет качества документации в общих метриках

## Примеры использования

### Пример 1: DORA-метрики для высокопроизводительной команды
Lead Time:
- Среднее: 1 день
- 85-й процентиль: 3 дня
- Цель: <= 2 дней для 85% изменений

Deployment Frequency:
- Текущее состояние: 10 раз в день
- Цель: >= 5 раз в день
- Тренд: стабильный рост

MTTR:
- Среднее: 1 час
- 85-й процентиль: 4 часа
- Цель: <= 2 часов

Change Failure Rate:
- Текущее: 5%
- Цель: <= 15%
- Тренд: снижение

### Пример 2: Метрики качества кода
Defect Density:
- Текущее: 0.02 дефекта на строку кода
- Сравнение: 0.05 в предыдущем проекте
- Цель: <= 0.03

Code Coverage:
- Unit тесты: 85%
- Integration тесты: 70%
- E2E тесты: 90% пользовательских сценариев
- Цель: >= 80% для критических путей

Technical Debt:
- Процент техдолга: 8%
- Время на исправление: 2 недели
- Цель: <= 10%

### Пример 3: Комплексная оценка качества
Quality Score:
- Формула: (Test Coverage × 0.3) + (Defect Density × 0.25) + (DORA Performance × 0.45)
- Текущий балл: 8.2 из 10
- Динамика: +15% за последние 6 месяцев

Risk Indicators:
- Количество критических уязвимостей: 0
- Плотность багов в проде: 0.001 на строку кода
- Время до обнаружения багов: 2 часа
- Скорость исправления: 4 часа
