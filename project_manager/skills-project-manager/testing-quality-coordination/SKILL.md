# Testing Quality Coordination

> **Meta:** v2.0.0 | 09-03-2026
> **Объединено из:** testing-coordination + uat-coordination

## Назначение

Комплексный навык для организации процесса тестирования и обеспечения качества программного продукта. Включает планирование тестирования, координацию приёмочного тестирования (UAT), управление дефектами и определение критериев готовности к релизу. Обеспечивает эффективное взаимодействие между командой разработки, тестирования и бизнес-пользователями.

## Когда использовать

Используйте этот навык:
- При планировании фазы тестирования проекта
- При координации работы команды тестирования
- При подготовке и проведении UAT
- При управлении процессом обнаружения и исправления дефектов
- При определении готовности продукта к релизу
- При сборе и анализе обратной связи от пользователей

## Функции

### 1. Планирование тестирования

Координация планирования тестирования:
- Определение уровней тестирования
- Планирование типов тестирования
- Установление сроков тестирования
- Выделение ресурсов для тестирования
- Определение критериев начала тестирования
- Определение критериев приёмки

### 2. Подготовка тестового окружения

Подготовка тестового окружения:
- Координация настройки окружения
- Обеспечение доступа к окружениям
- Поддержка релевантности данных
- Обеспечение изоляции окружений
- Планирование обновлений окружения

### 3. Координация UAT

Координация приёмочного тестирования:
- Определение участников UAT
- Планирование временных рамок
- Подготовка тестовых сценариев
- Определение критериев успеха
- Планирование поддержки пользователей

### 4. Управление дефектами

Управление отслеживанием дефектов:
- Настройка процесса регистрации
- Классификация и приоритизация дефектов
- Отслеживание статуса исправлений
- Анализ трендов дефектов
- Управление SLA по исправлениям

### 5. Определение критериев релиза

Определение критериев готовности к релизу:
- Определение порогов качества
- Установка критериев прохождения тестов
- Проверка соответствия требованиям
- Оценка рисков оставшихся дефектов
- Утверждение готовности к релизу

### 6. Обучение пользователей и сбор обратной связи

Подготовка пользователей и работа с обратной связью:
- Подготовка обучающих материалов
- Проведение обучения
- Подготовка документации для пользователей
- Организация сессий поддержки
- Создание каналов обратной связи

## Интеграция с System Analyst

- Учёт нефункциональных требований при планировании тестирования
- Использование сценариев от System Analyst для тестирования
- Проверка соответствия требованиям
- Привлечение System Analyst к анализу дефектов
- Обновление требований на основе результатов тестирования

## Примеры использования

### Пример 1: Координация тестирования веб-приложения

Типы тестирования:
- Модульные тесты: 100% покрытие критических путей
- Интеграционные тесты: проверка интеграции с внешними API
- E2E тесты: ключевые пользовательские сценарии
- Нагрузочное тестирование: до 1000 одновременных пользователей
- Безопасность: проверка уязвимостей OWASP Top 10

Окружения:
- Development: для разработчиков
- Staging: для интеграционного тестирования
- Pre-production: для приёмочного тестирования
- Production: для UAT и нагрузочного тестирования

Критерии релиза:
- 0 критических и 0 высоких дефектов
- 95% успешных автоматизированных тестов
- Время отклика не более 2 секунд под нагрузкой
- Как минимум 90% тест-кейсов пройдено

### Пример 2: Управление дефектами в проекте

Процесс:
- Регистрация: через JIRA
- Классификация: по типу и приоритету
- Назначение: ответственным разработчикам
- SLA: критические - 24 часа, высокие - 3 дня, средние - 1 неделя
- Статусы: Open → In Progress → Resolved → Closed

Отслеживание:
- Ежедневные обзоры статуса дефектов
- Еженедельные отчёты о трендах
- Метрики: плотность дефектов, время исправления
- Ретроспективы: анализ причин дефектов

### Пример 3: Планирование UAT

Планирование UAT:
- Участники: ключевые пользователи и бизнес-аналитики
- Сценарии: на основе пользовательских историй
- Инфраструктура: pre-production окружение
- Продолжительность: 2 недели
- Критерии успеха: 95% сценариев успешно завершено

Координация:
- Подготовка тестовых данных
- Обучение пользователей процессу тестирования
- Сбор и анализ обратной связи
- Итеративное улучшение продукта
- Подготовка заключения о готовности

### Пример 4: Критерии готовности к релизу

Чеклист готовности:
- [ ] Все критические сценарии работают корректно
- [ ] Требования пользователей выполнены
- [ ] Производительность соответствует SLA
- [ ] Безопасность соответствует стандартам
- [ ] Документация предоставлена

Утверждающие:
- Product Owner: бизнес-функциональность
- Security Officer: аспекты безопасности
- Compliance Manager: регуляторное соответствие
- Ключевые пользователи: юзабилити

Решение:
- Условия: не более 5 нерешённых комментариев высокого приоритета
- Время: 3 дня на принятие решения
- Процесс: электронное утверждение с комментариями
