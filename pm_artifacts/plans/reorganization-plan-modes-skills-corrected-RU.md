# Скорректированный план реорганизации: Документация по режимам и навыкам

## Обзор
Этот документ описывает скорректированный план реорганизации правил и навыков, связанных с режимами и навыками в проекте пользовательских режимов KiloCode. Цель состоит в том, чтобы перенести недостающую информацию из файлов правил в соответствующие навыки, а затем удалить избыточные файлы правил, чтобы избежать усложнения и дублирования контекста модели.

## Текущее состояние

### Файлы правил для анализа:
- `.kilocode/rules/Custom Modes.md` - исчерпывающее руководство по созданию пользовательских режимов
- `.kilocode/rules/Custom Rules.md` - информация о системе пользовательских правил
- `.kilocode/rules/kilocode-rules-instructions-en-v4.md` - рекомендации по созданию правил
- `.kilocode/rules/Prompt Engineering.md` - лучшие практики написания эффективных промптов
- `.kilocode/rules/Skills.md` - информация о реализации навыков агента
- `.kilocode/rules/Specification Agent Skills.md` - полная спецификация формата для навыков агента

### Навыки для обновления:
- `universal-coding-agent/skills-universal-coding-agent/create-custom-modes` - навык для создания пользовательских режимов
- `universal-coding-agent/skills-universal-coding-agent/kilo-skill-expert` - навык для создания/редактирования/отладки навыков

## План действий

### Фаза 1: Анализ и извлечение информации
1. Тщательно проанализировать каждый файл правил, чтобы выявить уникальную информацию, которая отсутствует в соответствующих навыках
2. Определить конкретные разделы и рекомендации, которые должны быть перенесены

### Фаза 2: Обогащение навыков недостающей информацией
1. **Обновить `create-custom-modes/SKILL.md` с информацией из:**
   - `Custom Modes.md`: детали конфигурации YAML, примеры, лучшие практики, разделы о регулярных выражениях
   - `Custom Rules.md`: информация о системе правил, форматировании и структуре
   - `Prompt Engineering.md`: лучшие практики для написания четких и конкретных инструкций
   - `kilocode-rules-instructions-en-v4.md`: структурные принципы, рекомендации по языку, алгоритмы создания правил

2. **Обновить `kilo-skill-expert/SKILL.md` с информацией из:**
   - `Skills.md`: детали реализации, преимущества, местоположения навыков
   - `Specification Agent Skills.md`: полная спецификация формата, структура каталогов, формат SKILL.md
   - `kilocode-rules-instructions-en-v4.md`: категории правил, лучшие практики, рекомендации по структуре

### Фаза 3: Проверка полноты информации
1. Убедиться, что вся важная информация из правил успешно перенесена в навыки
2. Проверить, что навыки теперь содержат полную документацию для выполнения задач
3. Убедиться, что структура навыков остается удобной для использования

### Фаза 4: Удаление избыточных файлов правил
1. После успешного обновления навыков удалить следующие файлы:
   - `.kilocode/rules/Custom Modes.md`
   - `.kilocode/rules/Custom Rules.md`
   - `.kilocode/rules/kilocode-rules-instructions-en-v4.md`
   - `.kilocode/rules/Prompt Engineering.md`
   - `.kilocode/rules/Skills.md`
   - `.kilocode/rules/Specification Agent Skills.md`

## Подробный план переноса информации

### 1. Из `Custom Modes.md` в `create-custom-modes/SKILL.md`:
- Подробные примеры конфигурации YAML
- Объяснение полей (slug, name, description, roleDefinition, groups, whenToUse, customInstructions)
- Рекомендации по использованию регулярных выражений (fileRegex)
- Лучшие практики и советы по работе с YAML
- Примеры конфигураций для разных типов режимов

### 2. Из `Custom Rules.md` в `create-custom-modes/SKILL.md`:
- Информация о структуре правил
- Форматирование и использование Markdown
- Примеры правил безопасности и ограничений доступа

### 3. Из `Prompt Engineering.md` в `create-custom-modes/SKILL.md`:
- Рекомендации по четкости и конкретике инструкций
- Советы по предоставлению контекста

### 4. Из `kilocode-rules-instructions-en-v4.md` в оба навыка:
- Структурные принципы (одна ответственность, иерархическая организация)

### 5. Из `Skills.md` в `kilo-skill-expert/SKILL.md`:
- Дополнительная информация о работе навыков в KiloCode
- Подробные сведения о местоположениях навыков
- Информация о приоритетах и переопределениях

### 6. Из `Specification Agent Skills.md` в `kilo-skill-expert/SKILL.md`:
- Полная спецификация формата SKILL.md
- Требования к полям (name, description)
- Информация о необязательных полях (license, compatibility, metadata)
- Рекомендации по структуре каталогов

## Ожидаемые результаты

### Улучшенная полнота навыков
- Навыки будут содержать всю необходимую информацию для выполнения задач
- Пользователи смогут использовать навыки как полноценные руководства
- Устранение необходимости обращаться к файлам правил для получения информации

### Упрощенная структура
- Удаление дублирующихся файлов упростит структуру проекта
- Снижение сложности контекста для модели
- Более четкое разделение ответственности между навыками и другими компонентами

### Повышенная эффективность
- Навыки станут самодостаточными и исчерпывающими
- Упрощение процесса обучения и использования
- Снижение вероятности противоречий между источниками информации

## Критерии успеха

1. Все важные сведения из файлов правил успешно перенесены в соответствующие навыки
2. Навыки остаются структурированными и удобными для использования
3. После обновления навыков соответствующие файлы правил удалены
4. Функциональность и полезность навыков повышены за счет дополнительной информации
5. Устранено дублирование информации между файлами правил и навыками
